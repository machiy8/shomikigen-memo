# 賞味期限管理Webアプリ MVP仕様書・Codex実装指示書

## 目的

賞味期限・消費期限が近い食品を管理し、食べ忘れ・期限切れ・食品ロスを防ぐためのシンプルなWebアプリを作成する。

まずは家族用・検証用のWebアプリとして開発する。  
Google Play公開やAndroidアプリ化は後工程とし、今回のMVPでは「登録」「一覧表示」「期限順ソート」「完了処理」「カテゴリ管理」を優先する。

---

## 開発方針

- まずはWebアプリとして動作させる
- スマホのブラウザで使いやすいUIにする
- Android Chromeでの利用を主対象にする
- データは最初はブラウザのLocalStorageに保存する
- ログイン機能は作らない
- サーバー・DBは使わない
- MVPなので、余計な機能は入れすぎない
- ただし後でAndroidアプリ化、PWA化、Firebase/Supabase連携しやすい構造にする

---

## MVPで実装する機能

### 1. 商品登録

登録項目は以下。

| 項目 | 必須 | 内容 |
|---|---|---|
| 商品名 | 必須 | 食品名、商品名を入力 |
| 賞味期限 | 必須 | 日付選択 |
| 数量 | 任意 | 数値入力。未入力なら1 |
| カテゴリ | 必須 | 初期カテゴリから選択 |
| メモ | 任意 | 自由入力 |
| 通知日数 | 任意 | MVPではUIだけ作ってもよい。実通知は後回し可 |

---

### 2. 初期カテゴリ

初期カテゴリは以下の5つ。

1. 食材
2. お菓子
3. 調味料
4. レトルト・保存食
5. 飲料

---

### 3. 一覧表示

登録された商品を一覧表示する。

一覧には以下を表示する。

- 商品名
- 賞味期限
- 残り日数
- 数量
- カテゴリ
- メモの有無またはメモ本文の一部
- 食べきったボタン
- 編集ボタン
- 削除ボタン

デフォルトの並び順は、賞味期限が近い順。

---

### 4. 残り日数表示

今日の日付を基準に、賞味期限までの日数を表示する。

表示例：

- 今日まで
- あと1日
- あと3日
- あと7日
- 期限切れ 2日

---

### 5. 期限状態の色分け

期限に応じてカードやラベルの見た目を変える。

目安：

| 状態 | 条件 | 表示 |
|---|---|---|
| 期限切れ | 今日より前 | 赤系、期限切れ表示 |
| 今日まで | 今日 | 赤系または強調 |
| 3日以内 | 残り1〜3日 | オレンジ系 |
| 7日以内 | 残り4〜7日 | 黄色系 |
| 余裕あり | 8日以上 | 通常表示 |

色はCSSで調整可能にする。

---

### 6. 食べきった処理

各商品に「食べきった」ボタンを付ける。

押した商品は、通常一覧から非表示にする。  
削除ではなく、`status` を `completed` に変更する。

データ上は残しておく。

---

### 7. 削除処理

完全に不要な商品は削除できるようにする。

削除時は確認ダイアログを出す。

例：

> この商品を削除しますか？

---

### 8. 編集処理

登録済みの商品を編集できるようにする。

編集可能項目：

- 商品名
- 賞味期限
- 数量
- カテゴリ
- メモ

---

### 9. フィルター

最低限、以下のフィルターを実装する。

- すべて
- 期限間近
- 期限切れ
- カテゴリ別
- 食べきった商品

「すべて」には `completed` は表示しない。  
「食べきった商品」では `completed` のみ表示する。

---

### 10. 検索

商品名で検索できるようにする。

検索対象：

- 商品名
- メモ

MVPでは部分一致でよい。

---

### 11. LocalStorage保存

登録データはLocalStorageに保存する。

保存キー例：

```txt
expiry_items
```

データ構造例：

```ts
type ExpiryItem = {
  id: string;
  name: string;
  expiryDate: string; // YYYY-MM-DD
  quantity: number;
  category: string;
  memo: string;
  notifyDaysBefore?: number;
  status: "active" | "completed";
  createdAt: string;
  updatedAt: string;
  completedAt?: string;
};
```

---

### 12. データの初期化

設定画面、または開発用ボタンとして、全データ削除機能を用意する。

削除前に必ず確認を出す。

## 通知機能について

MVPでは、実際のプッシュ通知は必須ではない。

ただし、将来的に通知を追加できるように、データ項目として `notifyDaysBefore` は持たせる。

初期UIでは以下の選択肢を用意する。

- 通知しない
- 当日
- 1日前
- 3日前
- 7日前
- 30日前

Web Push通知は後工程で検討する。  
Androidアプリ化する場合は、アプリ側のローカル通知で実装する。

## 実装時の注意点

- 賞味期限の日付比較は、時刻の影響を受けないように日付単位で処理する
- 今日の日付はユーザーのローカル日付を使う
- LocalStorageのデータが壊れていてもアプリが落ちないようにする
- 入力バリデーションを行う
- 商品名と賞味期限が未入力の場合は保存できないようにする
- 数量が未入力の場合は1にする
- 編集時は既存データをフォームに反映する
- 削除時は確認ダイアログを出す
- `completed` の商品は通常一覧に表示しない
- 期限近い順に自動ソートする
- コードは後でDB保存に変更しやすいよう、データ操作関数を分離する

- 商品を編集できる

- 食べきった商品を確認できる
- 食べきった商品を戻せる
- カテゴリ別に絞り込みできる

## Codexへの作業指示

この仕様に沿って、React + Vite + TypeScript で賞味期限管理WebアプリのMVPを実装してください。

優先順位は以下です。

1. 基本画面作成
2. 商品登録
3. LocalStorage保存
4. 一覧表示
5. 賞味期限順ソート

9. 編集・削除
10. フィルター
11. 検索
12. スマホ向けUI調整

実装後、以下を確認してください。

- `npm install`
- `npm run dev`
- ブラウザで動作確認
- 商品登録、編集、削除、食べきった処理ができること
- ページ再読み込み後もデータが残ること
- 期限順に並ぶこと
- 期限切れや期限間近が分かること

UIはシンプルで構いませんが、家族がスマホで使いやすい見た目にしてください。
